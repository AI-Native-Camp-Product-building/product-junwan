# Dashboard Finalization Plan

**작성일**: 2026-04-03  
**대상 프로젝트**: AdInsight  
**목표**: 현재 코드베이스를 기능정의서 기준의 실사용 가능한 MVP 상태로 마감한다.

---

## 1. 배경

현재 프로젝트는 다음 상태에 도달해 있다.

- Google Sheets -> Supabase 동기화 파이프라인이 동작한다.
- `ad_raw`, `ad_normalized`, `sheet_sync_latest` 기반 데이터 구조가 마련되어 있다.
- 홈 대시보드, 플랫폼별, 매체별 화면의 기본 구조가 존재한다.
- 날짜 범위 선택기, 추이 차트 탭, 인사이트 패널, CSV export, 히트맵 테이블 등 일부 Phase 1 기능이 선구현되어 있다.

반면 기능정의서 대비 아래 갭이 남아 있다.

- 메인 흐름에서 데이터 테이블이 실제로 빠져 있다.
- 글로벌 필터와 페이지별 고정 필터의 동작 정합성이 부족하다.
- KPI/지표 체계가 기능정의서보다 축약되어 있다.
- 비교 기간, 통화 토글, 목표별/작품별, 설정, SSO는 아직 미완성이다.
- 기능정의서 기준 10개 플랫폼과 현재 운영 데이터 계층의 8개 플랫폼 사이에 범위 차이가 있다.

이 문서는 위 상태를 바탕으로, 이번 마감 라운드에서 실제로 끝낼 항목을 정리한다.

## 2. 이번 라운드 목표

이번 라운드의 목표는 아래 한 문장으로 정의한다.

> 마케팅 기획자가 홈, 플랫폼별, 매체별 화면에서 날짜 범위를 바꿔가며 KPI, 추이, 상세 테이블, 인사이트를 일관된 방식으로 확인할 수 있는 수준까지 제품을 마감한다.

핵심 사용자 경험 목표:

- 30초 안에 전체 성과를 읽을 수 있다.
- 3클릭 안에 플랫폼별/매체별 상세 맥락으로 이동할 수 있다.
- 필터를 바꿔도 화면 구조와 데이터 반응이 예측 가능하다.
- export 가능한 수준의 상세 테이블이 항상 노출된다.

- 비교 기간 UI는 이번 라운드에서 넣지 않는다.

- 기능정의서의 용어는 최대한 반영하되, 현재 데이터 구조가 없는 지표는 프론트 계산 가능한 범위에서만 반영한다.
- 새 페이지를 늘리기보다 홈/플랫폼별/매체별 3개 화면 완성도를 높인다.

## 8. 리스크와 대응

### 리스크 1. 기능정의서와 현재 데이터 범위 차이

- 내용: 문서는 10개 플랫폼 기준, 현재 운영 데이터는 8개 플랫폼 기준
- 대응: 이번 계획서는 8개 플랫폼 운영 기준으로 마감하고, 10개 확장은 별도 작업으로 분리

### 리스크 2. 비교 기간 부재

- 내용: 기능정의서에는 직전 동일기간 비교가 중요하나 현재 UI는 없다
- 대응: 이번 라운드에서는 내부 계산 또는 안내 문구 수준으로만 정리하고, 정식 UI는 다음 단계로 미룸

### 리스크 3. lint/build 신뢰도 부족

- 내용: dashboard 외 폴더까지 lint 에러가 섞여 있다
- 대응: dashboard 범위 우선 정리 후, 배포에 영향을 주는 에러만 별도 분리 대응

### 리스크 4. 필터 상태 충돌

- 내용: 페이지 고정 필터와 공통 필터가 엉키면 UX가 불안정해진다
- 대응: URL 규칙과 필터 병합 규칙을 먼저 고정하고 구현
