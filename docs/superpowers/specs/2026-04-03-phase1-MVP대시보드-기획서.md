## 1. Phase 1 목표

> 기획자가 대시보드에 진입하면 30초 내에 "지금 잘 되고 있는가?"를 판단할 수 있고, 3클릭 내에 원인 분석에 도달할 수 있는 MVP를 완성한다.

### 설계 원칙 (기능정의서 v1.0 기반)

| 원칙 | 적용 |
|------|------|
| **30초 룰** | 홈 진입 시 KPI 4개 + 추이 차트가 즉시 보임 |
| **3클릭 룰** | 홈 → 플랫폼 선택 → 매체 필터 = 상세 데이터 도달 |
| **비교 우선** | 모든 KPI에 전기대비 변화율(%) 표시 |
| **예외 기반 어텐션** | 이상치에 컬러코딩 집중 (초록/노랑/빨강) |

## 3. Phase 1 범위 (P0만)

### 3.1 사이드바 재구성

**현재**: Overview / Data Explorer / Analysis (3개 고정 메뉴)
**목표**: 홈 / 플랫폼별 / 매체별 (3개 메뉴, 서브메뉴 없음)

```
홈           ← /dashboard           전체 통합 요약
플랫폼별     ← /dashboard/platform   페이지 내에서 플랫폼 Select로 쿼링
매체별       ← /dashboard/medium     페이지 내에서 매체 Select로 쿼링
```

- 사이드바는 **3개 메뉴만** — 서브메뉴로 8개 플랫폼을 나열하지 않음
- 플랫폼별 페이지 진입 시 상단에 **플랫폼 Select 드롭다운** 표시, 선택하면 해당 데이터로 쿼링
- 매체별 페이지도 동일 패턴 — **매체 Select 드롭다운**으로 전환
- 플랫폼 목록은 `sheet_source` 테이블에서, 매체 목록은 `medium_map`에서 동적 생성
- 현재 선택된 메뉴 하이라이트 + 좌측 인디케이터 바 (기존 패턴 유지)

### 3.2 날짜 범위 선택기

**현재**: 월 멀티셀렉트 (Select 컴포넌트)
**목표**: 3가지 모드의 날짜 선택기

```
[주별] [월별] [커스텀]     ← 탭 또는 세그먼트 컨트롤

주별 모드:   [◀ 이전주]  2026.03.24 ~ 03.30 (3월 4주차)  [다음주 ▶]
월별 모드:   [◀ 이전월]  2026년 3월                       [다음월 ▶]
커스텀 모드: [2026-01-15] ~ [2026-03-31]  ← 캘린더 팝오버로 시작일/종료일 선택
```

**주별 모드:**
- 좌우 화살표로 주 단위 이동
- 현재 주 표시: 날짜 범위 + 몇 주차
- 기본값: 이번 주

**월별 모드:**
- 좌우 화살표로 월 단위 이동
- 기본값: 이번 달

**커스텀 모드:**
- 시작일 / 종료일을 각각 캘린더 팝오버로 선택
- shadcn Calendar 컴포넌트 활용 (date-fns 기반)
- 범위 선택 후 즉시 데이터 갱신

**공통:**
- 필터 상태를 URL 쿼리 파라미터에 동기화 (`?mode=weekly&start=2026-03-24&end=2026-03-30`)
- API 쿼리를 날짜 범위 기반으로 전환
  - 현재: `months=2026-01,2026-02` 방식
  - 목표: `startDate=2026-01-01&endDate=2026-03-31` 방식
- 모든 뷰(홈/플랫폼별/매체별)에서 동일한 글로벌 날짜 컨트롤 사용

### 3.3 추이 차트 탭 전환

**현재**: ROAS Area Chart 1개 (전체 고정)
**목표**: 4개 탭 (광고비 / 가입 / 결제 / ROAS)

```
[광고비] [가입] [결제] [ROAS]     ← shadcn Tabs
  └─ 선택된 탭의 시계열 차트
     - X축: 월별 (또는 일별, granularity에 따라)
     - Y축: 해당 지표 값
     - 라인: 국가별 다중 라인 (필터에 따라)
```

- 현재 `ChartSection` 컴포넌트를 확장
- ROAS Area Chart의 기존 로직을 재사용하되, metric 파라미터로 분기
- 탭 전환 시 `animate-in fade-in` 트랜지션

### 3.4 플랫폼별 뷰

**경로**: `/dashboard/platform`  (단일 페이지, 동적 라우팅 아님)
**레이아웃**: 홈과 동일 구성 (KPI → 차트 → 테이블 → 인사이트)

- 페이지 상단에 **플랫폼 Select 드롭다운** — 8개 플랫폼 중 선택
- 선택하면 해당 플랫폼 데이터만 필터링하여 표시
- 공통 `DashboardShell` 재사용, `filters.countries`를 선택된 플랫폼으로 고정
- Select 값을 URL 쿼리 파라미터에 동기화 (`?platform=레진 KR`)
- 기본값: 첫 번째 플랫폼 또는 전체

### 3.5 매체별 뷰

**경로**: `/dashboard/medium`  (단일 페이지)
**레이아웃**: 홈과 유사하되, 행 기준이 국가별

- 페이지 상단에 **매체 Select 드롭다운** — Meta / YouTube / TikTok 등 선택
- 선택하면 해당 매체의 크로스 플랫폼 성과 표시
- KPI: 해당 매체 전체 합산
- 차트: 국가별 해당 매체 성과 비교
- 테이블: 국가를 행으로, 지표를 열로
- Select 값을 URL 쿼리 파라미터에 동기화 (`?medium=Meta`)

### 3.6 데이터 테이블 강화

**현재**: 기본 @tanstack/react-table
**추가 기능**:

1. **컬럼 정렬** — 클릭으로 오름차순/내림차순 토글 (이미 부분 구현)
2. **CSV 내보내기** — 테이블 상단에 "CSV 다운로드" 버튼
   - 현재 필터 상태가 반영된 데이터만 내보내기
   - 파일명: `adinsight-{뷰}-{날짜범위}.csv`
3. **히트맵 컬러코딩 토글** — 숫자 셀 배경에 값 크기 기반 색상
   - 열별 독립 스케일 (min-max 정규화)
   - 토글 버튼으로 ON/OFF

### 3.7 인사이트 패널

**위치**: 메인 콘텐츠 하단 (테이블 아래)
**내용**: 규칙 기반 자동 생성

1. **이번 기간 주요 변화 Top3** — 전기대비 증감률이 가장 큰 지표 3개
2. **이상치 알림** — CPA ±30%, ROAS ±30% 등 임계값 초과 시 알림
3. **컬러코딩**: 긍정(초록) / 주의(노랑) / 위험(빨강)

현재 `src/lib/insights.ts`에 규칙 기반 인사이트 로직이 이미 있음 (랜딩 페이지용). 이걸 대시보드 데이터에 맞게 확장.

## 4. 서브에이전트 구성 제안

### 구성: 3개 에이전트

| 에이전트 | 역할 | 담당 작업 |
|---------|------|----------|
| **Agent A: Frontend Architect** | 라우팅 + 레이아웃 구조 | 사이드바 재구성, 동적 라우팅 스캐폴딩, 공통 레이아웃 |
| **Agent B: Dashboard Components** | 차트/테이블/필터 컴포넌트 | 날짜 프리셋, 차트 탭 전환, 테이블 강화, CSV export |
| **Agent C: Data & Insights** | API 확장 + 인사이트 | 날짜 범위 API, 인사이트 패널, 히트맵 로직 |

### 실행 흐름

```
[Agent A] 라우팅 & 레이아웃     ← 선행 (Day 1-3)
  사이드바, 동적 라우팅, 공통 셸
          │
          ├──────────────────────────────┐
          ▼                              ▼
[Agent B] 컴포넌트             [Agent C] 데이터 & 인사이트
  차트 탭, 테이블, CSV           API 확장, 인사이트 패널
          │                              │
          └──────────┬───────────────────┘
                     ▼
            [통합 검증 + 브라우저 테스트]
```

- Agent A가 선행: 라우팅과 레이아웃이 있어야 B/C가 컴포넌트를 배치할 수 있음
- Agent B/C는 병렬: 컴포넌트와 데이터 레이어가 독립적

## 6. API 변경

### `/api/dashboard` 쿼리 파라미터 확장

현재:
```
GET /api/dashboard?countries=레진 KR,US&months=2026-01,2026-02&mediums=Meta&goals=결제
```

추가:
```
GET /api/dashboard?countries=레진 KR&startDate=2026-01-01&endDate=2026-03-31&mediums=Meta
```

- `startDate` / `endDate`가 있으면 `months` 무시
- `ad_normalized` 뷰의 `ad_date` 컬럼으로 범위 필터링
- `months`도 여전히 지원 (하위 호환)

### `/api/sidebar-data` (신규)

사이드바 메뉴 데이터를 서버에서 제공:

```json
{
  "platforms": [
    { "slug": "lezhin-kr", "name": "레진 KR", "currency": "KRW" },
    { "slug": "bomtoon-kr", "name": "봄툰 KR", "currency": "KRW" },
    ...
  ],
  "mediums": ["Meta", "YouTube", "TikTok", "Google GDN", ...]
}
```
