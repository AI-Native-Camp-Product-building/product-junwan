## 핵심 탭 6개 구조

### 1. 주간_로케일별

기준주 단위로 로케일별 성과 요약. 전주 대비 증감 포함.

**헤더:** 로케일, 광고비(현재주), 결제금액, ROAS, 회원가입, 결제전환, 가입CPA, 광고비(전주), ROAS(전주), 광고비증감%, ROAS변화p
**데이터 예시:** 봄툰KR, 14,509,753, 16011760, 110%, 2483, 2085, 5844, 15855759, 99%, -8.50%, ↑11p
**TOTAL 행:** 있음 (하단 합계)
**메타정보:** "기준주: 2026-03-16 ~ 2026-03-22 (3월3주차) | 전주: 2026-03-09 ~ 2026-03-15 | 단위: KRW"

→ **쿼리 방식:** ad_raw에서 `date_raw`를 주 단위로 GROUP BY, 로케일(sheet_name)별 SUM

### 2. 주간_매체별

기준주 단위로 로케일×매체별 성과.

**헤더:** 로케일, 매체, 광고비, 결제금액, ROAS, 회원가입, 결제전환, 가입CPA
**데이터 예시:** 봄툰KR, 유튜브, 8785756, 8125230, 92%, 1977, 1154, 4444
**정렬:** 광고비 내림차순

→ **쿼리 방식:** ad_raw에서 주 단위 필터 + GROUP BY sheet_name, medium_raw

### 3. 주간_목표별

기준주 단위로 로케일×목표별 성과.

**헤더:** 로케일, 목표, 광고비, 결제금액, ROAS, 회원가입, 결제전환, 가입CPA
**데이터 예시:** 봄툰KR, 가입, 9,747,150, 7872760, 81%, 2405, 1173, 4053
**정렬:** 광고비 내림차순

→ **쿼리 방식:** ad_raw에서 주 단위 필터 + GROUP BY sheet_name, goal_raw

### 4. 월간_로케일별

(빈 탭 — 아직 데이터 없음. 주간과 동일 구조이나 월 단위 집계로 추정)

→ **쿼리 방식:** ad_raw에서 month_raw로 GROUP BY sheet_name

### 5. 작품별_효율

두 섹션으로 구분:
- **■ 결제 캠페인 ROAS 상위 소재 (광고비 순)**
  - 헤더: 순위, 로케일, 작품명, 광고비, 결제금액, ROAS, 결제전환
  - 필터: goal = 결제 계열만
  - 정렬: ROAS 높은 순 (광고비 기준)
  
- **■ 가입 캠페인 CPA 소재 현황 (광고비 순)**
  - 헤더: 순위, 로케일, 작품명, 광고비, 회원가입, 가입CPA
  - 필터: goal = 가입 계열만
  - 정렬: 광고비 높은 순

→ **쿼리 방식:** ad_raw에서 주 단위 필터 + GROUP BY sheet_name, creative_name, goal 계열별 분리

### 6. 원본_마스터

주간보고 원본 데이터. 주간_매체별과 동일 구조 + 노출/CTR/비고 추가.

**헤더:** 로케일, 매체, 광고비(KRW), 결제금액(KRW), ROAS, 회원가입, 결제전환, 가입CPA, 노출, CTR, 비고
**하단:** 데이터 수집 현황 메모

→ **쿼리 방식:** 주간_매체별과 동일 + impressions, ctr 추가

## 공통 패턴

1. **시간 단위:** "주간"이 핵심 — ISO week 기반 집계 필요
2. **집계 차원:** 로케일, 매체, 목표, 작품명 — 4개 축
3. **지표:** 광고비, 결제금액, ROAS(=결제금액/광고비), 회원가입, 결제전환, 가입CPA(=광고비/회원가입)
4. **비교:** 전주 대비 증감 (광고비증감%, ROAS변화pp)
5. **정렬:** 대부분 광고비 내림차순
6. **KRW 통일:** 모든 금액은 원화 기준

## UI에서 재현하려면

사용자가 선택해야 하는 것:
- **시간 축:** 주간 / 월간
- **주차 선택:** 달력 or 주차 피커 (예: "3월 3주차")
- **집계 축:** 로케일별 / 매체별 / 목표별 / 작품별
- **추가 필터:** 국가, 매체, 목표 (선택적)

→ 이 조합으로 ad_raw를 GROUP BY 쿼리하면 종합 시트의 모든 탭을 재현 가능
