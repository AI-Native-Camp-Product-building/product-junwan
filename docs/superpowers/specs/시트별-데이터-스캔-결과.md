## 6. 데이터 품질 관찰 사항

### 6.1 FORMATTED_VALUE vs UNFORMATTED_VALUE

스캔은 `FORMATTED_VALUE` 모드로 실행했기 때문에 숫자가 문자열로 반환됨.
실제 파이프라인(`sheets-client.ts`)은 `UNFORMATTED_VALUE`를 사용하므로 숫자 파싱 문제 없음.

그러나 다음 시트에서 **통화 기호가 셀에 포함**되어 있음:
- **FR**: 광고비에 `€` 접두사 (`€9.39`), 원화에 `₩` 접두사 (`₩15,963`), 결제금액에 `€` 접두사
- **ES**: 광고비에 `$` 접두사 (`$58.04`)

`UNFORMATTED_VALUE` 모드에서는 이 기호가 제거될 수 있으나, **셀 서식이 아닌 실제 텍스트로 입력된 경우** `parseNumeric()`이 실패할 수 있음. 확인 필요.

### 6.2 `.` (dot) placeholder

TH 시트에서 빈 값 대신 `.`을 사용:
- 월별: `.`
- 일자: `.`
- 매체: `.`
- 목표: `.`
- 광고비: `.`

`parseNumeric()`은 `.`을 null로 처리함 (이미 구현됨). `parseString()`은 `.`을 문자열 `.`로 반환하므로, medium_map과 goal_map에 `.`이 포함되어야 함 (이미 포함됨).

### 6.3 `#DIV/0!` 에러 값

FR 시트의 가입CPA 컬럼에 `#DIV/0!` 값 존재. `parseNumeric()`에서 NaN -> null로 처리됨.

### 6.4 `-` (하이픈) placeholder

DE, TW 시트의 가입CPA 컬럼에 `-` 값 존재. `parseNumeric()`에서 `-`를 null로 처리함 (이미 구현됨).

### 6.5 TH 시트 데이터량 극소

TH는 7행만 존재. 광고가 최근에 시작된 것으로 추정. 파이프라인은 문제없이 동작할 것이나, 리포팅 시 통계적으로 의미 있는 데이터는 아님.

### 6.6 200행 제한

레진 KR, 봄툰 KR, US, FR은 200행 제한에 걸렸으므로 실제 데이터가 더 많을 수 있음.
전체 데이터 규모는 `totalRowsInSheet` - `headerRow`로 추정 가능:
- 레진 KR: ~200행
- 봄툰 KR: ~200행
- US: ~200행
- FR: ~200행
- DE: 131행
- TW: 193행
- ES: 81행
- TH: 7행
