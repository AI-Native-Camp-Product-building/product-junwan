# 260428\_로케일 상세 페이지 및 사이드바 고도화 계획

## Summary

- 새 라우트는 `/dashboard/locale`로 만들고, URL은 `?locale=US&start=...&end=...`처럼 query 기반으로 운영한다.
- 사이드바의 `탐색` 아래에 `로케일` 펼침 메뉴를 추가하고, 하위에 8개 로케일을 표시한다.
- 오버뷰의 로케일 카드는 클릭 가능하게 바꾸고, 현재 날짜/매체/목표 필터를 유지한 채 `/dashboard/locale`로 이동한다.
- HTML 참고 파일의 로케일 상세 구조를 AmInsight 디자인 시스템으로 재구성하되, 모든 수치/추이/인사이트는 현재 선택 필터에 따라 동적으로 계산한다.
- 주차별 그래프는 HTML과 시각 디자인은 다르게 하되, 목적과 형태는 동일하게 유지한다.

## Key Changes

- 사이드바는 `홈`, `탐색`, `로케일`, `피드백` 순서로 표시하고, `로케일`은 기본 펼침 메뉴로 둔다.
- 로케일 상세 화면은 기존 `/api/dashboard`의 `AdRow[]`를 사용하고, 국가 필터를 선택 로케일로 잠근다.
- 상세 페이지의 집계는 선택 기간만 기준으로 하며 오버뷰의 비교기간 확장 로직은 사용하지 않는다.
- 상세 콘텐츠는 상단 KPI/인사이트와 `주차별`, `작품별`, `소재별`, `매체별` 탭으로 구성한다.
- 주차별 탭은 HTML과 동일하게 광고비+CTR, 회원가입+ROAS+가입 CPA 복합 차트와 주차별 표를 제공한다.

## Public Interfaces

- 새 URL:
  - `/dashboard/locale?locale=US`
  - `/dashboard/locale?locale=봄툰%20KR&start=2026-04-01&end=2026-04-19`
- 기존 API route, Supabase schema, `QueryDefinition`은 변경하지 않는다.
- 기존 `/dashboard/platform`은 삭제하지 않고 병행 유지한다.

## Test Plan

- `npm run typecheck`
- 변경 파일 대상 `npx eslint ...`
- 가능하면 `npm run build`
- 사이드바 로케일 하위 항목 클릭 시 올바른 `locale` query로 이동하는지 확인한다.
- 홈에서 날짜/매체/목표 필터를 바꾼 뒤 로케일 카드를 클릭하면 해당 필터가 상세 페이지에 유지되는지 확인한다.
- `US`, `봄툰 KR`, `레진 KR`, `TW`에서 KPI, 인사이트, 차트, 테이블 값이 필터 변경에 따라 갱신되는지 확인한다.
- 데이터 없는 기간에서 빈 상태가 깨지지 않는지 확인한다.

## Assumptions

- 로케일 표기는 현재 DB/필터 옵션 값 그대로 사용한다.
- 사이드바 목록은 8개 고정 순서로 시작하며, 데이터가 없는 로케일도 메뉴에는 표시한다.
- HTML의 고정 문구는 그대로 복사하지 않고, 같은 의사결정 구조를 현재 필터 데이터 기반의 동적 템플릿으로 재작성한다.
- 주차별 그래프는 Chart.js가 아닌 현재 프로젝트의 Recharts/shadcn chart 체계로 구현한다.
