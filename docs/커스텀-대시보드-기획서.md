# 커스텀 대시보드 시스템 기획서

## 1. 배경 및 목표

### 현재 문제

- 홈(Overview)과 탐색(Explore)이 같은 데이터/차트/비교 로직을 각각 구현
- 고정 레이아웃 — 유저가 원하는 정보 조합을 볼 수 없음
- 기능 중복으로 유지보수 비용 증가

### 목표

유저가 **자연어 또는 직접 조작**으로 대시보드를 자유롭게 구성하는 위젯 기반 시스템.
기능(쿼리 엔진)은 하나로 통합, 노출(UI)만 유저가 결정.

---

## 2. 시스템 아키텍처

### 2.1 데이터 레이어

```
자연어 입력 → Gemini → WidgetConfig JSON
                          ↓
위젯 설정 (query_config) → query-engine.ts → dynamic_aggregate RPC → 집계 결과
                          ↓
위젯 렌더러 (차트/테이블/KPI) → 화면 표시
```

기존 `dynamic_aggregate` RPC를 그대로 사용. 새로운 집계 로직 불필요.

### 2.2 DB 스키마

```sql
-- 유저별 대시보드
CREATE TABLE user_dashboards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_email TEXT NOT NULL,
  name TEXT NOT NULL DEFAULT '내 대시보드',
  is_default BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_email, name)
);

-- 대시보드 내 위젯
CREATE TABLE dashboard_widgets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dashboard_id UUID NOT NULL REFERENCES user_dashboards(id) ON DELETE CASCADE,
  widget_type TEXT NOT NULL,          -- 'kpi' | 'line' | 'area' | 'donut' | 'table' | 'comparison'
  title TEXT NOT NULL,
  query_config JSONB NOT NULL,        -- {dimensions, metrics, filters, dateRange, compare}
  display_config JSONB DEFAULT '{}',  -- {chartType, showLegend, colors, curveType}
  grid_position JSONB NOT NULL,       -- {x, y, w, h}
  sort_order INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

`query_config`는 기존 `QueryDefinition` 타입과 동일 — 변환 레이어 불필요.

### 2.3 API 라우트

| 경로                                 | 메서드       | 용도                         |
| ------------------------------------ | ------------ | ---------------------------- |
| `/api/dashboards`                    | GET/POST     | 대시보드 목록/생성           |
| `/api/dashboards/[id]`               | PATCH/DELETE | 대시보드 수정/삭제           |
| `/api/dashboards/[id]/widgets`       | GET/POST     | 위젯 목록/추가               |
| `/api/dashboards/[id]/widgets/[wid]` | PATCH/DELETE | 위젯 수정/삭제               |
| `/api/dashboards/[id]/widgets/batch` | PATCH        | 드래그 후 위치 일괄 업데이트 |
| `/api/ai-widget`                     | POST         | 자연어 -> 위젯 설정 생성     |

## 3. 프론트엔드 아키텍처

### 3.1 파일 구조

```
src/widgets/
  types.ts             -- WidgetConfig, DashboardLayout, WidgetType
  registry.ts          -- 위젯 타입 -> 컴포넌트 + 기본 크기 매핑
  widget-shell.tsx     -- 공통 래퍼 (Card + 드래그 핸들 + 메뉴)
  widget-renderer.tsx  -- config -> fetch -> 렌더링 파이프라인
  use-widget-query.ts  -- WidgetConfig.query -> useExploreQuery 어댑터
  layout-store.ts      -- Zustand: 레이아웃 + 위젯 상태 관리
  widgets/
    kpi-widget.tsx     -- KPI 카드 위젯
    trend-widget.tsx   -- 추이 라인/영역 차트 위젯
    donut-widget.tsx   -- 도넛 차트 위젯
    table-widget.tsx   -- 데이터 테이블 위젯
    compare-widget.tsx -- 비교 차트 위젯
```

### 3.2 핵심 타입

```typescript
type WidgetType = "kpi" | "trend" | "donut" | "table" | "comparison";

interface WidgetConfig {
  id: string;
  type: WidgetType;
  title: string;
  query: {
    dimensions: DimensionKey[];
    metrics: MetricKey[];
    filters: FilterCondition[];
    dateRange: DateRange | null;
    compare?: CompareConfig;
  };
  display?: {
    chartType?: "line" | "area" | "bar";
    showLegend?: boolean;
    curveType?: "linear" | "monotone" | "natural" | "basis";
  };
}

interface LayoutItem {
  i: string; // = WidgetConfig.id
  x: number;
  y: number;
  w: number; // 12컬럼 그리드 기준
  h: number;
  minW?: number;
  minH?: number;
}
```

### 3.3 위젯 레지스트리

```typescript
const WIDGET_REGISTRY = {
  kpi: { component: KpiWidget, defaultLayout: { w: 3, h: 2 } },
  trend: { component: TrendWidget, defaultLayout: { w: 6, h: 3 } },
  donut: { component: DonutWidget, defaultLayout: { w: 3, h: 3 } },
  table: { component: TableWidget, defaultLayout: { w: 12, h: 4 } },
  comparison: { component: CompareWidget, defaultLayout: { w: 6, h: 3 } },
};
```

### 3.4 레이아웃 엔진

`react-grid-layout`의 `ResponsiveGridLayout` 사용.

- 12/8/4 컬럼 반응형 (lg/md/sm)
- 모바일(<768px): 1컬럼 스택, 편집 비활성
- Zustand store -> localStorage 캐시 + Supabase 동기화

### 3.5 성능 최적화

- **쿼리 중복 제거**: 동일 query_config 해시 → 캐시 공유
- **뷰포트 렌더링**: IntersectionObserver로 화면 밖 위젯 지연 로드
- **Suspense 경계**: 위젯별 독립 로딩 상태

## 4. UX 설계

### 4.1 첫 방문 유저

1. `/dashboard`에 **프리셋 레이아웃** 표시 (현재 KPI + 차트 + 테이블)
2. 상단 배너: "이 대시보드를 자유롭게 커스터마이즈할 수 있어요" + "편집 시작"
3. 배너는 localStorage로 1회 표시 후 숨김

### 4.2 편집 모드

**토글 방식**: 헤더에 "편집" / "완료" 버튼

편집 모드 진입 시:

- 그리드 점선 표시
- 각 위젯에 드래그 핸들(좌상단) + 메뉴(우상단: 설정/복제/삭제)
- 리사이즈 핸들(우하단 모서리)
- 상단 툴바 확장: "위젯 추가", "템플릿", "되돌리기"
- 하단 AI 프롬프트 바 확장

편집 종료: "완료" 클릭 → 자동 저장 (Notion 스타일, 별도 저장 버튼 없음)

### 4.3 위젯 추가

"위젯 추가" 클릭 → **우측 사이드바 카탈로그** 열림

카테고리:

- KPI 카드 (광고비, ROAS, 가입수, CPA...)
- 차트 (추이선, 막대, 도넛, 비교)
- 테이블 (매체 요약, 국가 요약, 상세 데이터)

각 항목: 썸네일 + 한줄 설명. 클릭하면 그리드 하단에 드롭.

### 4.4 위젯 설정

위젯 메뉴 "설정" 또는 **더블클릭** → 인라인 팝오버 패널

설정 항목:

- 데이터: 차원/지표 드롭다운 (Explore 쿼리빌더 컨트롤 재사용)
- 필터: 국가/월/매체 (위젯별 독립)
- 기간: 위젯별 독립 dateRange
- 표시: 차트 타입 토글, 범례, 제목 편집

변경 시 실시간 반영 (적용 버튼 없음).

### 4.5 AI 대시보드 생성

하단 중앙에 **플로팅 프롬프트 바** (Spotlight 스타일)

- 편집 모드: 항상 표시
- 뷰 모드: 아이콘으로 축소

입력 예시: "지난 달 국가별 광고비 추이를 월별로 보여줘"

AI 응답 → 위젯 자동 생성 + 그리드 배치 (300ms 스태거 애니메이션)
결과는 완전 편집 가능 — AI는 시작점.

### 4.6 템플릿

헤더 메뉴 "템플릿에서 시작" → 모달 갤러리

프리셋:

- 마케팅 오버뷰 (현재 홈과 동일)
- 국가별 비교 분석
- 매체 퍼포먼스
- 빈 대시보드

## 5. 마이그레이션 전략

### Phase 1: 위젯 인프라 (1주)

- DB 테이블 생성
- 위젯 타입/레지스트리/셸 구현
- react-grid-layout 통합
- 기존 차트 컴포넌트를 위젯으로 래핑

### Phase 2: 기본 기능 (1주)

- 위젯 추가/삭제/드래그/리사이즈
- 위젯 설정 패널 (인라인 팝오버)
- 편집 모드 토글
- Supabase 저장/로드

### Phase 3: AI 통합 + 프리셋 (1주)

- `/api/ai-widget` — Gemini -> WidgetConfig
- 플로팅 프롬프트 바
- 프리셋 템플릿 (시드 데이터)
- 첫 방문 온보딩 배너

### Phase 4: 기존 페이지 통합 (선택)

- 현재 `/dashboard` 를 커스텀 대시보드로 교체
- `/dashboard/explore` 는 "전체화면 탐색 위젯"으로 유지 가능
- 플랫폼별/매체별 페이지는 프리셋 템플릿으로 대체

## 6. 기술 선택 요약

| 항목        | 선택                                       | 이유                                              |
| ----------- | ------------------------------------------ | ------------------------------------------------- |
| 레이아웃    | react-grid-layout                          | 드래그/리사이즈 + 반응형, 가장 성숙한 라이브러리  |
| 상태관리    | Zustand                                    | 경량, persist 미들웨어로 localStorage 자동 동기화 |
| 위젯 데이터 | 기존 query-engine + useExploreQuery        | 새 집계 로직 불필요                               |
| AI          | Gemini (기존)                              | 프롬프트 확장만으로 WidgetConfig 생성 가능        |
| 저장        | Supabase user_dashboards/dashboard_widgets | RLS로 유저별 격리                                 |
| 차트        | Recharts (기존)                            | 위젯으로 래핑만                                   |
