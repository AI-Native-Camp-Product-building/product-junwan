## Phase 1: 코드 정리

중복되어 있던 컴포넌트와 유틸리티를 정리했습니다.

- 같은 기능의 필터바가 3개 → 1개로 통합
- 날짜 선택기 2개 → 1개로 통합
- KPI 카드 2개 → 1개로 통합
- 숫자/통화 포맷 함수가 6개 파일에 흩어져 있던 것 → 1개 파일로 통합
- 사용하지 않는 Explorer 페이지 제거

**사용자 화면 변화: 없음** (내부 코드만 정리)

---

## Phase 2: 쿼리 엔진 구축

데이터를 자유롭게 조회할 수 있는 백엔드 엔진을 만들었습니다.

- 어떤 조합이든 "차원 + 지표"를 선택하면 서버에서 즉시 집계해서 반환
- 기존: 8,000행을 전부 브라우저로 보내서 계산 → 느림
- 변경: 서버(DB)에서 계산 후 결과만 전송 → 수십 ms 이내 응답
- 비교 기능(기간 비교, 항목 비교) 엔진도 포함

**사용자 화면 변화: 없음** (보이지 않는 백엔드)

---

## Phase 3: 탐색(Explore) 페이지

핵심 기능입니다. Google Analytics / Amplitude 스타일의 데이터 탐색 도구를 만들었습니다.

### 직접 선택 모드
클릭만으로 쿼리를 조합하는 문장형 UI:
```
[광고비, ROAS] 를 [국가] 별로 나눠서 보여줘
+ 필터 추가
기간: 📅 2026.04.01 (수) ~ 📅 2026.04.06 (월)
```

### AI 모드
자연어로 질문하면 AI가 쿼리를 자동 생성:
- "렌탈걸즈랑 하렘의남자 4월 ROAS 비교해줘"
- "Meta 매체에서 이번달 작품별 가입수 높은 순으로"
- 예시 프롬프트 버블 클릭으로 바로 실행 가능

### 비교 모드
- **기간 비교**: 3월 vs 4월 같은 기간 대비 변화율 확인
- **항목 비교**: 작품A vs 작품B, 국가A vs 국가B 직접 비교

### 결과 테이블
- 컬럼 추가/제거 칩: 결과를 보면서 "월", "일자", "매체" 등 차원을 클릭 한번으로 추가/제거 → 즉시 재조회
- 정렬: 컬럼 헤더 클릭으로 오름/내림차순
- CSV 내보내기

### 네비게이션
- 사이드바 + 상단 헤더에 "탐색" 메뉴 추가

## 추가 작업 (Phase 외)

### Google Workspace SSO
- 더미 로그인 폼 → Google OAuth 로그인으로 교체
- @lezhin.com 도메인만 접속 허용
- 미인증 시 대시보드 접근 차단 → 로그인 페이지로 리다이렉트
- 사이드바에 사용자 프로필 + 로그아웃 버튼

### 데이터 정합성 수정
- CTR / ROAS: 시트의 비율값(0.18)을 퍼센트(18%)로 변환
- 광고비: 봄툰 KR의 "원화" 컬럼 데이터 오류 대응 (자동 fallback)
- 동기화 방식: upsert → delete + insert로 변경 (시트에서 삭제된 행이 DB에 남는 문제 해결)
- 달력 네비게이션 수정 (월 이동 불가 → 정상 동작)

## 남은 작업

### Phase 4: 리포트 페이지
- 자주 쓰는 쿼리를 프리셋으로 저장
- "주간 로케일별", "작품별 ROAS 랭킹" 등 기본 리포트 제공
- 기존 Analysis 페이지(6개 고정 탭)를 대체

### Phase 5: Overview 경량화
- 홈 대시보드도 서버 집계 방식으로 전환 (현재 8,000행 전송 → 집계 결과만 전송)
- 플랫폼별/매체별 페이지를 Explore로 통합

### Phase 6: 최종 정리
- 미사용 코드/라우트 삭제
- 사이드바 최종 구성: 홈 / 탐색 / 리포트

## 현재 페이지 구조

```
/login              → Google 로그인 (SSO)
/dashboard          → Overview (KPI + 차트 + 상세 테이블)
/dashboard/explore  → 탐색 (쿼리 빌더 + AI 모드 + 비교)
/dashboard/platform → 플랫폼별 성과 (추후 Explore로 통합 예정)
/dashboard/medium   → 매체별 성과 (추후 Explore로 통합 예정)
/dashboard/analysis → 분석 (추후 리포트로 대체 예정)
```
