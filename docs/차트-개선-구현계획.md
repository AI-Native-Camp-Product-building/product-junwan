# 차트 개선 구현 계획

## Context

홈 대시보드 추이 차트에서 0값이 "데이터 미입력"인 경우에도 라인이 0까지 내려가는 문제.
탐색 추이 차트에서 곡선 완만함을 유저가 직접 조절할 수 있는 슬라이더 요청.

---

## Task 1: 홈 추이 차트 — 0값 미입력 구간 라인 끊기

### 원인

`computeTrendData()` (dashboard-shell.tsx:145-191)에서 국가별 집계 시 adSpend/revenue/signups가 모두 0이어도 `point[country] = 0`으로 설정. Recharts가 0을 실제 데이터로 렌더링.

### 해결 방향

- 집계 시 **raw 데이터가 있었는지** 추적 (`hasData` 플래그)
- 세 지표(adSpend, revenue, signups) 모두 0이면 = 미입력 -> `undefined` 반환
- 하나라도 양수면 = 실제 데이터 -> 계산값(0 포함) 유지
- Recharts Area는 `connectNulls` 기본 false -> undefined 구간에서 자연스럽게 끊김

### 변경 파일

- `src/types/dashboard.ts` — TrendPoint 타입에 `undefined` 허용
- `src/components/dashboard/dashboard-shell.tsx` — computeTrendData() hasData 로직

## Task 2: 탐색 추이 차트 — 곡선 완만도 슬라이더

### 설계

슬라이더 0~3, 4단계 discrete:

| 값  | Recharts type | 설명                |
| --- | ------------- | ------------------- |
| 0   | `linear`      | 직선, 변화 다이나믹 |
| 1   | `monotone`    | 기본값 (현재)       |
| 2   | `natural`     | 자연스러운 곡선     |
| 3   | `basis`       | 매우 완만한 베지어  |

UI: 차트 하단 중앙, 작은 슬라이더 + 양쪽 라벨 ("직선 <-> 곡선")

### 변경 파일

- 신규: `src/components/ui/slider.tsx` — @base-ui/react Slider 래핑
- `src/components/explore/explore-chart.tsx` — 곡선 슬라이더 통합
