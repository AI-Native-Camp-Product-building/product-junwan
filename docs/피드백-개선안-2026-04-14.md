### FB-1. 기간별 추이 차트 명칭 변경 + 자세히 보기

**피드백 원문:**

> 홈 > overview > 성과 추이 > 기간별 광고비 추이
>
> 1. 광고비 / 가입자수 / 가입cpa / 결제금액 / roas 로 명칭 변경
> 2. 자세히 보기 추가

**현재 상태 (trend-chart.tsx):**

| 현재 탭 명칭 | 요청 명칭 | 변경 필요     |
| ------------ | --------- | ------------- |
| 광고비       | 광고비    | 동일 ✅       |
| 가입         | 가입자수  | 변경 필요     |
| 결제         | 결제금액  | 변경 필요     |
| ROAS         | ROAS      | 동일 ✅       |
| (없음)       | 가입 CPA  | **신규 추가** |

**개선 방안:**

1. **명칭 변경** (단순 수정):
   - `trend-chart.tsx` TAB_CONFIG에서:
     - `가입` → `가입자수`
     - `결제` → `결제금액`
   - 차트 description도 동일 변경

2. **가입 CPA 탭 추가**:
   - TAB_CONFIG에 `signupCpa` 키 추가
   - 값: `adSpend / signups` (mapRow에서 계산)
   - 포맷: `formatKrw`

3. **"자세히 보기" 링크 추가**:
   - 각 탭 차트 하단에 "탐색에서 자세히 보기 →" 링크
   - 클릭 시 `/dashboard/explore`로 이동, 해당 지표 pre-select
   - 기존 `explore-link.ts` 유틸 활용

**난이도:** 낮음 | **파일:** `trend-chart.tsx`, `explore-link.ts`

### FB-2. 국가별 성과 요약 테이블 명칭 변경

**피드백 원문:**

> 홈 > overview > 성과 추이 > 국가별 성과 요약
> 광고비 / roas / 가입자 / 결제금액 / 가입 cpa로 명칭 변경

**현재 상태 (country-summary-table.tsx):**

| 현재 컬럼 헤더 | 요청 명칭 | 변경 필요     |
| -------------- | --------- | ------------- |
| 광고비         | 광고비    | 동일 ✅       |
| ROAS           | ROAS      | 동일 ✅       |
| 가입           | 가입자    | 변경 필요     |

| 결제금액       | 결제금액  | 동일 ✅       |
| (없음)         | 가입 CPA  | **신규 추가** |

2. **가입 CPA 컬럼 추가:**
   - 값: `adSpend / signups`
   - 포맷: `formatKrw`

3. **결제전환 컬럼 처리:**
   - 피드백에서 "결제전환" 언급 없음 → 유저가 불필요하다고 판단한 듯
   - 사용자에게 확인 후 제거 또는 유지 결정

**난이도:** 낮음 | **파일:** `country-summary-table.tsx`

### FB-3. 작품별 카테고리 추가

**피드백 원문:**

> 1. 작품별 상세 데이터 추가
> 2. 작품별 검색 시, 광고비와 성과에 대한 그래프 노출

**분석:**

현재 홈 대시보드에 "작품별 Top 10" 차트(`creative-ranking.tsx`)가 있지만, 작품 단위 상세 뷰(개별 작품 drill-down)는 없음.

**개선 방안:**

1. **작품별 전용 페이지 또는 섹션:**
   - 옵션 A: `/dashboard/creative` 신규 페이지
   - 옵션 B: 기존 Explore 페이지에서 `creative_name` 차원으로 안내

2. **작품 검색 + 상세 그래프:**
   - 작품명 검색 → 해당 작품의 기간별 광고비/가입/결제/ROAS 추이 차트
   - Explore 페이지의 비교 모드(항목 비교)로 이미 부분 구현됨
   - 작품 선택 시 자동으로 Explore로 이동 + 필터 pre-set이 가장 현실적

3. **추천 접근:**
   - 홈 "작품별 Top 10" 차트에서 작품명 클릭 → Explore 페이지로 이동
   - URL 파라미터: `dims=date&metrics=ad_spend_krw,signups,revenue_krw,roas&filters=[{"field":"creative_name","operator":"eq","value":"작품명"}]`
   - 기존 `explore-link.ts` 유틸 확장

**난이도:** 중간 | **파일:** `creative-ranking.tsx`, `explore-link.ts`

### FB-4. 플랫폼별 비교 필터

**피드백 원문:**

> 플랫폼별 > 플랫폼별 성과
> 플랫폼 비교 필터 추가 (ex. 레진 kr / 봄툰 kr 비교)

**분석:**

현재 플랫폼별 뷰(`/dashboard/platform`)는 국가를 하나씩 보는 구조. 두 플랫폼을 나란히 비교하는 기능 없음.

**개선 방안:**

1. **옵션 A: Explore 비교 모드 활용 (빠름)**
   - 플랫폼별 페이지에서 "비교하기" 버튼 추가
   - 클릭 시 Explore 페이지 비교 모드로 이동
   - URL: `compare=item&dimension=country&baseValue=레진 KR&compareValue=봄툰 KR`
   - 이미 구현된 항목 비교 기능 재사용

2. **옵션 B: 플랫폼별 페이지 내 비교 UI (완성도 높음)**
   - 필터에 "비교 플랫폼" 셀렉터 추가
   - 두 플랫폼 KPI/차트를 좌우 배치
   - 개발량 많음 — Phase 2 추천

**추천:** 옵션 A 먼저 (버튼 하나 추가), 사용자 반응 보고 옵션 B 검토

**난이도:** 낮음(A) / 높음(B) | **파일:** platform 페이지 컴포넌트

### FB-5. KPI 카드 비교 기간 사용자 지정

**피드백 원문 + 첨부 이미지 분석:**

> 홈 > 오버뷰 > 데이터 비교 수치를 전월 대비 x > 사용자가 지정한 대로 비교 가능하게끔 설정해야함

**첨부 이미지 분석:**

![KPI 카드](feedback-images/img5) 스크린샷에서 확인:

- KPI 카드: 총 광고비 ₩3255만, 총 회원가입 6,171, 총 결제전환 5,529
- 각 카드에 "전월 대비 📈" + 변화율(+92.6%, +115.3%)
- 기간: 2026-03-30 ~ 2026-04-05

**현재 문제:** 비교 기준이 **전월 동기간 고정**. 사용자가 비교 대상 기간을 선택할 수 없음.

**개선 방안:**

1. **비교 기간 선택 UI 추가:**
   - KPI 카드 영역 상단에 비교 모드 셀렉터:
     - `전월 대비` (현재, 기본값)
     - `전주 대비`
     - `전년 동기 대비`
     - `직접 설정` → 커스텀 날짜 범위 피커
   - 선택한 비교 기간으로 changeRate 재계산

2. **구현 방식:**
   - `kpi-cards-refined.tsx`에 비교 모드 state 추가
   - `computeKpiSummary()` 함수에 비교 기간 파라미터 전달
   - 대시보드 API에서 비교 기간 데이터도 함께 요청 (또는 별도 요청)

3. **참고:** 이미지에서 광고비가 `₩3255만`으로 표시 → #11 수정(원 단위)과 함께 자동 해결됨

**난이도:** 중간 | **파일:** `kpi-cards-refined.tsx`, `dashboard-queries.ts`

### FB-6. 탐색 작품별 CTR 컬럼 추가

**피드백 원문 + 첨부 이미지 분석:**

> 탐색 > 작품별 > ctr 카테고리 추가 필요

**첨부 이미지 분석:**

![Explore 테이블](feedback-images/img6) 스크린샷에서 확인:

- 탐색 페이지에서 작품명(creative_name) 차원으로 쿼리 실행한 결과
- 현재 표시 컬럼: 작품명, 광고비, 결제금액, ROAS
- CTR이 지표 선택에 포함되어 있지 않은 상태

**분석:**

CTR은 이미 `query-schema.ts`에 지표로 존재함 (`ctr` key). 사용자가 쿼리 빌더에서 CTR을 추가하지 않은 것일 수 있음. 하지만 이번 세션에서 구현한 **지표 chips**로 해결 가능:

- 결과 테이블 위 지표 chips에서 `[+ CTR]` 클릭하면 바로 추가됨

**추가 개선:**

- 기본 지표 프리셋에 CTR 포함 검토
- 또는 "작품별 분석" 프리셋 쿼리 저장: `dims=creative_name, metrics=ad_spend_krw,revenue_krw,roas,ctr`

**상태:** 이미 구현된 기능으로 대부분 해결. 사용자 안내 또는 프리셋 추가만 필요.

**난이도:** 매우 낮음 | **파일:** 프리셋 데이터 또는 사용자 안내
