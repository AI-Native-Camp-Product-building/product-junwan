# AmInsight 디자인 시스템 가이드

## 1. 현재 문제

- 대시보드 컴포넌트 20개 파일, 72곳에서 다크 전용 하드코딩 스타일 사용
- `bg-white/[0.03]`, `rgba(255,255,255,0.08)`, `border-white/[0.06]` 등
- shadcn CSS 변수(`:root`, `.dark`)가 있지만 대시보드에서 무시하고 직접 값 사용
- 라이트/다크 전환 불가능

## 2. 디자인 원칙

### Gumroad에서 배우는 것

- **데이터가 주인공** — UI 크롬은 최대한 투명하게
- **볼드 숫자** — KPI는 크고 굵게, 즉시 인지
- **컬러 절약** — 차트 데이터에만 색상, 나머지는 무채색
- **충분한 여백** — 정보 과부하 방지

### 라이트/다크 공통 원칙

- 모든 스타일은 **시맨틱 토큰**으로 — 하드코딩 색상 금지
- 카드 배경은 **불투명** — 데이터 가독성 최우선
- glassmorphism은 **장식 요소에만** 제한 (사이드바, 모달 배경)
- 차트/테이블 영역은 solid 배경

## 3. 토큰 설계

### 3.1 신규 시맨틱 토큰

```css
:root {
  /* === 기존 shadcn 토큰 유지 === */

  /* === 대시보드 전용 토큰 (신규) === */

  /* 표면 (Surface) — 카드, 위젯 배경 */
  --surface: 0 0% 100%; /* 순백 */
  --surface-foreground: 0 0% 6%; /* 거의 블랙 */
  --surface-muted: 0 0% 97%; /* 연회색 — 중첩 카드, 테이블 행 */
  --surface-raised: 0 0% 100%; /* 떠있는 카드 (shadow로 구분) */

  /* 보더 */
  --border-subtle: 0 0% 90%; /* 카드 외곽선 */
  --border-muted: 0 0% 94%; /* 내부 구분선 */

  /* 텍스트 계층 */
  --text-primary: 0 0% 6%; /* 제목, KPI 숫자 */
  --text-secondary: 0 0% 35%; /* 부제, 설명 */
  --text-tertiary: 0 0% 55%; /* 보조 정보, 라벨 */

  /* 대시보드 크롬 (사이드바, 헤더) */
  --chrome: 0 0% 98%;
  --chrome-foreground: 0 0% 20%;
  --chrome-border: 0 0% 92%;
  --chrome-hover: 0 0% 95%;
  --chrome-active: 0 0% 92%;

  /* 위젯 */
  --widget-bg: 0 0% 100%;
  --widget-border: 0 0% 90%;
  --widget-shadow: 0 0% 0% / 0.04; /* 미세 그림자 */
  --widget-hover-shadow: 0 0% 0% / 0.08;

  /* 상태 */
  --success: 152 60% 42%;
  --warning: 38 92% 50%;
  --danger: 0 72% 51%;
  --info: 220 70% 55%;

  /* 브랜드 악센트 */
  --accent-brand: 220 70% 55%; /* AmInsight 블루 */
}

.dark {
  /* 표면 */
  --surface: 0 0% 7%;
  --surface-foreground: 0 0% 95%;
  --surface-muted: 0 0% 10%;
  --surface-raised: 0 0% 9%;

  /* 보더 */
  --border-subtle: 0 0% 15%;
  --border-muted: 0 0% 12%;

  /* 텍스트 */
  --text-primary: 0 0% 95%;
  --text-secondary: 0 0% 70%;
  --text-tertiary: 0 0% 50%;

  /* 크롬 */
  --chrome: 0 0% 5%;
  --chrome-foreground: 0 0% 80%;
  --chrome-border: 0 0% 12%;
  --chrome-hover: 0 0% 10%;
  --chrome-active: 0 0% 14%;

  /* 위젯 */
  --widget-bg: 0 0% 9%;
  --widget-border: 0 0% 15%;
  --widget-shadow: 0 0% 0% / 0.2;
  --widget-hover-shadow: 0 0% 0% / 0.3;

  /* 상태 (다크용 밝기 조정) */
  --success: 152 55% 50%;
  --warning: 38 85% 55%;
  --danger: 0 65% 55%;
  --info: 220 65% 60%;
}
```

```css
  --color-text-primary: hsl(var(--text-primary));
  --color-text-secondary: hsl(var(--text-secondary));
  --color-text-tertiary: hsl(var(--text-tertiary));
  --color-chrome: hsl(var(--chrome));
  --color-chrome-foreground: hsl(var(--chrome-foreground));
  --color-widget-bg: hsl(var(--widget-bg));
  --color-widget-border: hsl(var(--widget-border));

```

## 4. 컴포넌트 스타일 매핑

### 4.1 교체 규칙

| 하드코딩 (현재)           | 시맨틱 토큰 (목표)     |
| ------------------------- | ---------------------- |
| `bg-white/[0.03]`         | `bg-widget-bg`         |
| `border-white/[0.08]`     | `border-widget-border` |
| `backdrop-blur-[12px]`    | 제거 (불투명 배경)     |
| `text-white/90`           | `text-text-primary`    |
| `text-white/50`           | `text-text-secondary`  |
| `text-white/30`           | `text-text-tertiary`   |
| `bg-white/[0.04]` (hover) | `bg-chrome-hover`      |
| `rgba(100,149,237,0.9)`   | `text-info`            |
| `glass-card` 클래스       | `widget-card` 클래스   |

### 4.2 카드 스타일

```tsx
// Before (다크 전용)
<Card className="bg-white/[0.03] border-white/[0.08] backdrop-blur-[12px]">

// After (라이트/다크 자동)
<Card className="bg-widget-bg border-widget-border shadow-[0_1px_3px_hsl(var(--widget-shadow))]">
```

### 4.3 사이드바

```tsx
// Before
className = "bg-white/[0.02] border-r border-white/[0.06]";

// After
className = "bg-chrome border-r border-chrome-border";
```

### 4.4 KPI 숫자

```tsx
// Before
className = "text-2xl font-semibold";

// After (Gumroad 스타일 — 더 크고 굵게)
className = "text-3xl font-bold tabular-nums text-text-primary";
```

### 4.5 위젯 카드 (커스텀 대시보드용)

```css
.widget-card {
  background: hsl(var(--widget-bg));
  border: 1px solid hsl(var(--widget-border));
  border-radius: var(--radius-lg);
  box-shadow: 0 1px 3px hsl(var(--widget-shadow));
  transition: box-shadow 200ms ease;
}

.widget-card:hover {
  box-shadow: 0 4px 12px hsl(var(--widget-hover-shadow));
}

/* 편집 모드 */
.widget-card[data-editing] {
  border-style: dashed;
  border-color: hsl(var(--info));
}
```

## 8. 마이그레이션 계획

### Phase 1: 토큰 정의 (globals.css)

- `:root`와 `.dark`에 신규 토큰 추가
- `@theme inline`에 Tailwind 매핑 추가
- `.glass-card` -> `.widget-card` 교체

### Phase 2: 대시보드 크롬 교체 (사이드바 + 헤더)

- `dashboard-sidebar.tsx` — 하드코딩 -> 시맨틱
- `dashboard-header-refined.tsx` — 하드코딩 -> 시맨틱
- 파일 2개, 약 10곳

### Phase 3: 카드/차트 컴포넌트 교체

- KPI, TrendChart, DonutChart, 테이블 등
- `bg-white/[0.03]` -> `bg-widget-bg`
- `border-white/[0.08]` -> `border-widget-border`
- `backdrop-blur` 제거
- 파일 20개, 약 72곳

### Phase 4: 테마 토글 추가

- 헤더에 라이트/다크 토글 버튼
- `localStorage` + `prefers-color-scheme` 연동
- `<html>` 태그에 `class="dark"` 토글

### Phase 5: 위젯 시스템에 적용

- 신규 위젯 컴포넌트는 처음부터 시맨틱 토큰만 사용
- widget-shell에 `.widget-card` 클래스 적용

## 9. 하지 않는 것

- Liquid Glass / 과도한 glassmorphism — 데이터 가독성 저하
- 그라디언트 배경 — 차트 위 가독성 문제
- 컬러 과다 사용 — 차트 데이터에만 색상
- 다크 모드에서 순수 검정 (#000) — 눈 피로, `hsl(0 0% 7%)` 사용
- 라이트 모드에서 순수 흰색 (#FFF) 텍스트 위 — 대비 부족
