# 260428_AmInsight 브랜드명 및 오버뷰 로케일 카드 추가 계획

## 개요

- 쿼리 시스템, Supabase 스키마, API contract는 변경하지 않는다.
- 사용자에게 노출되는 주요 브랜드 표기는 `AdInsight`에서 `AmInsight`로 변경한다.
- 오버뷰의 `성과 추이 / 매체별 성과` 차트 바로 아래에 `로케일별 성과` 카드를 추가한다.
- 로케일 카드는 이번 단계에서 클릭 비활성 상태로 둔다. 플랫폼별 상세 이동은 후속 작업으로 분리한다.

## 구현 범위

- 브랜드명은 UI 텍스트, 메타데이터 title, export 파일명, 주요 운영 문서에 한정해 변경한다.
- 내부 코드 주석, SQL 코멘트, docker service명, 레거시 spec/plan 문서의 과거 히스토리 표기는 무리하게 전역 치환하지 않는다.
- 신규 `LocalePerformanceCards` 컴포넌트는 기존 `AdRow[]`와 `isLoading`만 받아 현재 오버뷰 필터가 적용된 데이터 기준으로 집계한다.
- 카드 지표는 광고비, 광고비 순위/비중, CTR, 결제 ROAS, 가입 CPA, 회원가입으로 구성한다.
- 카드 레이아웃은 모바일 1열, 태블릿 2열, 데스크톱 4열을 기본으로 한다.

## 집계 기준

- 광고비 비중: 로케일 광고비 / 현재 필터 결과 전체 광고비.
- CTR: 로케일 클릭 / 로케일 노출수.
- 결제 ROAS: `결제`, `첫결제` 목표 행의 결제금액 / 광고비.
- 가입 CPA: `가입` 목표 행의 광고비 / 회원가입. 가입 목표 데이터가 없고 `가입&결제` 데이터가 있으면 혼합 기준으로 fallback 표시한다.
- 회원가입: 해당 로케일의 전체 회원가입 합계.

## 검증 계획

- `rg "AdInsight|ADINSIGHT" src docs AGENTS.md README.md`로 잔여 표기를 확인한다.
- 의도적으로 남긴 잔여 표기는 내부 주석, 레거시 문서, 과거 피드백 원문 같은 비사용자 노출 항목으로 분류한다.
- `npm run lint`, `npm run typecheck`, 필요 시 `npm run test`를 실행한다.
- `/dashboard`에서 필터 변경 시 로케일 카드 수치가 동일한 필터 기준으로 갱신되는지 확인한다.
- 빈 데이터와 로딩 상태에서 카드 영역이 깨지지 않는지 확인한다.

## 잔여 표기 예외

- `src/types/*`, `src/lib/*`, `src/config/query-schema.ts`, `src/data/dummy-data.ts`의 파일 헤더 주석은 내부 개발용 표기라 이번 사용자 표시 변경 범위에서 제외한다.
- `src/lib/scan-sheets.ts`의 CLI 출력 문구는 내부 스캔 스크립트용이라 제외한다.
- `docs/피드백-수집-2026-04-14.md`의 `adinsight` 표기는 과거 피드백 원문 인용이라 보존한다.
- 이 문서의 변경 설명과 검증 명령에 포함된 기존 브랜드명 언급은 작업 기록 목적이라 보존한다.

## 후속 작업

- 로케일 카드 클릭 시 플랫폼별 상세 페이지로 이동하는 링크를 별도 작업으로 구현한다.
- Overview의 클라이언트 집계 영역을 장기적으로 `dynamic_aggregate` 기반 서버 집계로 이관한다.
