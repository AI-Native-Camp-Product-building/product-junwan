# 탐색 컬럼 필터 — 쿼리 빌더 연동 기획서

## 배경

현재 탐색(Explore) 페이지는 쿼리 빌더(상단)와 결과 테이블(하단)이 분리되어 있음.
사용자가 결과를 보다가 추가 필터링을 하려면 다시 상단 쿼리 빌더로 올라가서 조작해야 함.

**목표:** 결과 테이블 컬럼 헤더에서 직접 필터/정렬을 조작할 수 있게 하고, 쿼리 빌더와 양방향 동기화.

---

## 핵심 컨셉

```
┌─────────────────────────────────────────┐
│  쿼리 빌더 (QuerySentence)              │  ← 기존 문장형 빌더
│  [광고비, ROAS] 를 [국가] 별로 보여줘    │
│  필터: 국가 = 레진 KR, 봄툰 KR          │  ← 컬럼 필터 변경 시 자동 반영
└─────────────────────────────────────────┘
         ↕ 양방향 동기화
┌─────────────────────────────────────────┐
│  결과 테이블                             │
│  ┌──────────┬──────────┬──────────┐     │
│  │ 국가 ▼▲ │ 광고비 ▼▲│ ROAS ▼▲  │     │  ← 헤더 클릭: 정렬 + 필터 드롭다운
│  │ [필터]   │ [범위]   │ [범위]   │     │
│  ├──────────┼──────────┼──────────┤     │
│  │ 레진 KR  │ 59,277.. │  182.3%  │     │
│  │ 봄툰 KR  │ 12,345.. │   95.1%  │     │
│  └──────────┴──────────┴──────────┘     │
└─────────────────────────────────────────┘
```

## 기능 상세

### 1. 컬럼 헤더 인터랙션

헤더 클릭 시 드롭다운 팝오버가 열림. 컬럼 타입에 따라 내용 분기.

#### 텍스트 컬럼 (차원: 국가, 매체, 작품명, 목표, 소재종류 등)

```
┌──────────────────────┐
│ 국가                  │
│ ───────────────────── │
│ ⬆ 오름차순            │
│ ⬇ 내림차순            │
│ ───────────────────── │
│ 🔍 검색               │
│ ☑ 전체 선택           │
│ ☑ 레진 KR             │
│ ☑ 봄툰 KR             │
│ ☐ US                  │
│ ☐ DE                  │
│ ☐ FR                  │
│ ...                   │
│ ───────────────────── │
│ [적용]     [초기화]   │
└──────────────────────┘
```

- 체크박스 멀티 선택
- 검색 입력으로 값 필터링 (목록이 긴 경우)
- "전체 선택" 토글
- 적용 클릭 시 → `filters`에 `in` 조건 추가 → 쿼리 재실행
- 초기화 → 해당 컬럼 필터 제거

#### 숫자 컬럼 (지표: 광고비, 결제금액, ROAS, CTR, 노출수 등)

```
┌──────────────────────┐
│ 광고비                │
│ ───────────────────── │
│ ⬆ 오름차순            │
│ ⬇ 내림차순            │
│ ───────────────────── │
│ 범위 필터             │
│ 최소: [________]      │
│ 최대: [________]      │
│ ───────────────────── │
│ [적용]     [초기화]   │
└──────────────────────┘
```

- min/max 범위 입력
- 둘 다 입력: `gte` + `lte` 조건
- 하나만 입력: `gte` 또는 `lte` 단독
- 적용 → 쿼리 빌더 필터에 반영 + 재실행

#### 날짜 컬럼 (일자)

```
┌──────────────────────┐
│ 일자                  │
│ ───────────────────── │
│ ⬆ 오름차순            │
│ ⬇ 내림차순            │
│ ───────────────────── │
│ 시작: [____-__-__]    │
│ 종료: [____-__-__]    │
│ ───────────────────── │
│ [적용]     [초기화]   │
└──────────────────────┘
```

- 쿼리 빌더의 dateRange와 동기화

### 2. 양방향 동기화

#### 컬럼 필터 → 쿼리 빌더

컬럼에서 필터 적용 시:

1. 쿼리 빌더의 `filters` 배열에 조건 추가/수정
2. 이미 같은 필드 필터 있으면 **덮어쓰기** (중복 방지)
3. 자동 재실행 (`execute()` 호출)
4. 쿼리 빌더 UI에 반영된 필터 표시

#### 쿼리 빌더 → 컬럼 필터

쿼리 빌더에서 필터 변경 시:

1. 해당 컬럼 헤더에 필터 활성 표시 (아이콘 변경, 색상 등)
2. 드롭다운 열었을 때 현재 필터 값 pre-fill
3. 쿼리 빌더에서 필터 제거 → 컬럼 필터도 초기화

#### 동기화 규칙

- 동일 필드에 대한 필터는 **하나만 유지** (쿼리 빌더든 컬럼이든 마지막 변경 우선)
- 컬럼 필터는 쿼리 빌더 필터의 **뷰** — 별도 상태 아님
- 필터 소스는 항상 `useExploreQuery` 훅의 `filters` 배열

### 3. 정렬

- 현재: 헤더 클릭으로 클라이언트 정렬 (TanStack Table 내장)
- 변경: 헤더 클릭은 드롭다운 열기, 드롭다운 내 정렬 버튼으로 변경
- 정렬은 서버사이드 (`sort` 파라미터) + 클라이언트 폴백
- 정렬 상태도 쿼리 빌더의 `sort` 와 동기화

### 4. 필터 활성 표시

필터가 적용된 컬럼 헤더에 시각적 표시:

```
│ 국가 🔽🔵 │ 광고비 🔽 │ ROAS 🔽🔵 │
```

- 🔽: 드롭다운 가능 (기본)
- 🔵: 필터 활성 상태 (색상 도트 또는 아이콘 변경)

실제 구현은 아이콘이 아닌 CSS:

- 기본: `text-muted-foreground`
- 필터 활성: `text-primary` + 작은 도트 인디케이터

### 5. 차원 추가/제거 (기존 chips 개선)

현재 QueryResultTable에 dimension chips 있음. 유지하되:

- 추가 가능한 차원 목록에서 클릭 → 쿼리에 차원 추가 + 재실행
- 기존 차원 X 클릭 → 차원 제거 + 재실행
- 이 동작은 현재와 동일 (변경 불필요)

## 컴포넌트 구조

### 신규 컴포넌트

```
src/components/explore/column-filter-popover.tsx
```

- props: `column`, `type` (text|number|date), `currentFilter`, `onApply`, `onClear`, `onSort`
- 내부: 타입별 UI 분기 렌더링
- Popover 기반 (shadcn Dialog 아님 — 테이블 위에 가볍게 뜨는 형태)

### 수정 컴포넌트

```
src/components/explore/query-result-table.tsx
```

- 컬럼 헤더를 ColumnFilterPopover 트리거로 변경
- 필터 상태를 쿼리 빌더 filters에서 읽기
- 필터 변경 콜백을 쿼리 빌더로 전달

```
src/hooks/use-query.ts
```

- `addOrUpdateFilter(field, operator, value)` 함수 추가
- `removeFilter(field)` 함수 추가
- 필터 변경 시 자동 재실행 옵션

## 데이터 플로우

```
사용자가 컬럼 헤더 클릭
  → ColumnFilterPopover 열림
  → 값 선택 또는 범위 입력
  → "적용" 클릭
  → addOrUpdateFilter() 호출
  → useExploreQuery.filters 업데이트
  → 쿼리 빌더 UI 자동 반영 (QuerySentence)
  → execute() 자동 호출
  → 결과 테이블 갱신
  → 컬럼 헤더 필터 활성 표시
```

## 필터 값 로딩

### 텍스트 컬럼 체크박스 목록

- **현재 결과 기반**: 테이블에 표시된 데이터의 unique 값 목록
- **전체 목록 필요 시**: `/api/filters` 에서 이미 로드된 filterValueOptions 활용
  - country, month, medium, goal → 이미 있음
  - creative_name, creative_type → 현재 결과에서 추출 (전체 목록은 너무 많을 수 있음)

### 숫자 컬럼 범위

- 현재 결과의 min/max를 placeholder로 표시
- 유저가 직접 입력

---

## 구현 순서

1. **useExploreQuery 확장** — `addOrUpdateFilter`, `removeFilter`, 자동 재실행
2. **ColumnFilterPopover** — 텍스트/숫자/날짜 타입별 UI
3. **QueryResultTable 헤더 수정** — 드롭다운 트리거 + 필터 활성 표시
4. **양방향 동기화 검증** — 쿼리 빌더 ↔ 컬럼 필터 연동 테스트

## 리스크 & 고려사항

| 항목             | 내용                                                                        |
| ---------------- | --------------------------------------------------------------------------- |
| 자동 재실행 부하 | 필터 적용마다 API 호출. 디바운스 불필요 (명시적 "적용" 버튼)                |
| 값 목록 크기     | creative_name 수백 개 가능 → 검색 + 가상 스크롤 고려                        |
| 파생 지표 필터   | ROAS, CTR 등은 서버 집계 후 계산값 → 서버 필터 불가, 클라이언트 필터로 처리 |
| Popover 위치     | 테이블 스크롤 시 위치 이탈 방지 → Popover `side="bottom" align="start"`     |
| 모바일           | 현재 대시보드는 데스크톱 중심 → 모바일 대응 불필요                          |
